    👋 Welcome to DexaMinds!

    Welcome to the engineering team! This guide will help you get set up and productive as quickly as possible.

    🚀 First Week Checklist

    Day 1: Setup & Orientation

    • [ ] Complete HR onboarding
    • [ ] Set up your development environment
    • [ ] Get access to essential tools
    • [ ] Meet your onboarding buddy
    • [ ] Review team structure and roles

    Day 2-3: Technical Setup

    • [ ] Clone repositories
    • [ ] Install dependencies
    • [ ] Run the application locally
    • [ ] Set up your IDE
    • [ ] Configure Git and SSH keys

    Day 4-5: First Tasks

    • [ ] Complete starter tasks
    • [ ] Make your first PR
    • [ ] Set up 1:1 with your manager
    • [ ] Join team standups
    • [ ] Explore the codebase

    🛠 Development Environment

    Prerequisites

    • Node.js (v18+)
    • Git
    • Docker
    • VS Code or your preferred IDE

    Setup Steps

    1. Clone the repository bash git clone git@github.com:dexaminds/dexaminds.git cd dexaminds

    2. Install dependencies bash npm install

    3. Environment variables bash cp .env.example .env # Update values as needed

    4. Start the application bash npm run dev

    Codebase Navigation

    • tests/ - Test files
    • docs/ - Documentation
    • scripts/ - Utility scripts

    🤝 Team Practices

    Communication

    • Slack: Primary communication tool
    • Email: For formal communications
    • Notion: Documentation and wikis
    • GitHub: Code collaboration

    Development Workflow

    1. Create a feature branch
    2. Make your changes
    3. Write tests
    4. Open a PR
    5. Address review comments
    6. Get approval
    7. Merge to main

    Team Rituals

    • Daily Standup: 10:00 AM
    • Sprint Planning: Every other Monday
    • Retrospective: End of sprint
    • Tech Talks: Monthly

    🆘 Getting Help

    1. Ask your onboarding buddy
    2. Search documentation
    3. Check Slack channels
    4. Post in #help-engineering
    5. Schedule a pairing session

    🎯 First 30 Days Goals

    • Complete onboarding tasks
    • Make your first production commit
    • Understand the product architecture
    • Build relationships with team members
    • Identify areas for improvement
